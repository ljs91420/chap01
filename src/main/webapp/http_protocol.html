프로토콜 (Protocol)

  • 통신 규칙
  • 컴퓨터들 간의 통신 절차를 정하고, 해당 절차에 이름을 붙이는 것
  • ftp, http, https, telnet, smtp ... 등등

HTTP 프로토콜(http://)의 작동 순서

  1. 사용자(클라이언트)가 웹 브라우저의 주소창에 보고 싶은 웹 페이지의 주소(URL)를 입력한다.(링크를 클릭하는 것도 주소창에 URL을 입력하는 것과 같다.)
  2. 웹 브라우저는 URL을 해석하여 서버의 IP 주소를 알아낸 뒤 해당 서버로 요청(request)을 보낸다.
  3. 요청을 받은 웹 서버는 클라이언트가 보낸 요청에 담겨 있는 정보들로 알맞은 응답을 생성하여 응답(response)한다.
  4. 요청을 보낸 클라이언트의 웹 브라우저가 응답을 받아 알맞게 해석하여 클라이언트에게 그려서 보여준다.

URL

https:// comic.naver.com (:443) /webtoon/detail?titleId=828292&no=2&week=tue

프로토콜

  • URL의 //앞은 해당 url이 어떤 프로토콜을 사용하는지 나타내는 곳이다.
  • http 프로토콜을 사용하는 경우 기본 포트번호가 80번이다.
  • https 프로토콜을 사용하는 경우 기본 포트번호가 443번이다.

IP주소

  • 각 컴퓨터를 찾아가기 위한 주소
  • 0.0.0.0 ~ 255.255.255.255까지 사용할 수 있다.(IPv4)
  • 나중에 IPv4가 고갈되면 사용할 예정인 IPv6도 준비되어 있지만 아직 사용하지 않는다.
  • URL의 //부터 첫 번째 /가 나오기 전까지는 IP주소를 적는 곳이다.
  • IP주소 대신 comic.naver.com 같은 것을 적을 수도 있는데, 이는 IP주소를 대체하는 도메인 이름(Domain Name Service)이다.
  • 127.0.0.1은 현재 본인의 컴퓨터를 가리키는 루프백 IP주소이다.
  • localhost도 현재 본인의 컴퓨터를 가리키는 도메인 이름이다.

DNS 주소

  • IP주소 자리에 IP주소 대신 사용할 수 있는 보기 좋은 이름
  • 사용자가 도메인 이름을 사용해 요청을 보내는 경우 먼저 DNS 서버에 들러 해당 도메인을 IP주소로 치환한 후 요청을 보내게 된다.
  • DNS서버는 인터넷 서비스 제공자(ISP)들이 관리한다.(KT, SKT, LG ... 등)

포트 번호

  • 컴퓨터의 각 프로그램들이 데이터를 전송받기 위해 기다리는 곳
  • 포트번호는 0 ~ 65535번까지 사용할 수 있다.
  • 웹 서버는 전통적으로 80번 포트를 열고 기다리는 것이 고착화되어 있다.
  • 고정적으로 사용하는 잘 알려진 포트번호(well-known ports)들이 많이 존재한다.
  • 80: http, 443: https, 21: ftp, 23: telnet, ...

URI

  • URL에서 / 이후에 나오는 모든 내용을 URI라고 부른다.
  • 해당 서버에서 어떤 자원을 요청할 것인지 적는 곳이다.
  • 웹 서버는 URI에 담긴 정보를 통해 어떤 웹 페이지를 응답할지 결정할 수 있다.
